## Why

OpenPlant is a plant companion app, but its visual identity uses a black-on-white / white-on-navy monochrome scheme with a blue accent (#3171EC) — no green in the primary palette. This feels mismatched for a plant-focused app. Additionally, several areas of the UI have contrast problems: hardcoded light/dark colors in widgets don't always align with the active theme, and the absence of explicit Material 3 color slots (onPrimary, primaryContainer, onSurface, etc.) means derived colors are silently generated by Material, which can clash with manually-specified colors.

## What Changes

- **Change the color scheme from blue-accent/navy to a green-based palette.** The primary seed color becomes green (plant-appropriate). The accent/secondary color becomes a complementary green tone instead of blue.
- **Fix light/dark contrast throughout the widget layer.** Eliminate hardcoded color literals (`Color.fromRGBO(...)`) in widget files — replace with theme tokens so every widget properly responds to light/dark mode.
- **Define explicit Material 3 color slots** (primaryContainer, onPrimary, surface, onSurface, surfaceVariant, outline, tertiary) in `themes.dart` for both light and dark themes, so derived colors are predictable and contrast is guaranteed.
- **Audit and fix all contrast-violating spots.** Ensure no text-on-surface combination falls below WCAG AA contrast ratio.
- **Consolidate duplicated hardcoded color values** across widget files into theme tokens.

## Capabilities

### New Capabilities
- `app-theme`: Core theme system — color palette (green-based), Material 3 color scheme definitions for light and dark modes, text theme, and component theme overrides. This is a foundational capability that all pages depend on.

### Modified Capabilities

No existing feature specs have requirement changes — the theme is a cross-cutting visual concern that affects all pages implicitly. Implementation details (how each widget consumes theme tokens) belong in the design artifact, not in individual feature specs.

| `lib/pages/home/home_page.dart` | Update system UI overlay colors to match new palette |
| `lib/pages/home/widgets/bottom_nav_bar*.dart` | Use theme tokens for active/inactive icon colors |
| `lib/pages/home/widgets/side_nav_bar*.dart` | Use theme tokens for active/inactive colors |
| All page files with inline color usage | Migrate to theme tokens where colors are hardcoded |
